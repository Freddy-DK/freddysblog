It is kind of embarrassing that I write a post about how to connect to NAV Web Services from Windows Mobile 6, when Windows Mobile 6.5 has been out for almost half a year (that’s how much a gadget person I am:-))

I just downloaded the 6.5 SDK from [here](http://www.microsoft.com/downloads/details.aspx?displaylang=en&FamilyID=20686a1d-97a8-4f80-bc6a-ae010e085a6e) and tried out the exact same application as I wrote for Windows Mobile 6 in [this post](/2010/01/22/connecting-to-nav-web-services-from-windows-mobile-6/) and everything seems to work just fine, so please follow the steps in [this post](/2010/01/22/connecting-to-nav-web-services-from-windows-mobile-6/) to see how to connect from Windows Mobile 6.5 to NAV Web Services.

![image\_2 (2)](/assets/images/2010/connecting-to-nav-web-services-from-windows-mobile-6-5/image_2-2.png)

# Is it faster?

My tests on 6.5 is only on the emulator – but as far as I can see, it is definitely faster. The 1.5 second from Windows Mobile 6 is now down to 0.9 second and 0.8 second with PreAuthenticate set to true.

I will still do some more investigations on performance from Windows Mobile Web Services.
